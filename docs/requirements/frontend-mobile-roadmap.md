# Kids Challenge Frontend Mobile Roadmap

이 문서는 현재 `frontend-mobile` 구현 상태를 기준으로, 앞으로 필요한 화면/기능을 우선순위에 따라 정리한 실행 계획서다.

목표는 다음과 같다.

- 이미 만들어진 부모/아이 앱의 핵심 흐름을 실제 서비스 수준으로 완성한다.
- 더미 화면, 임시 문구, 미연결 기능을 순차적으로 제거한다.
- 각 단계가 끝날 때마다 실제 사용 가능한 기능 단위로 검증한다.

## 1. 현재 구현 상태 요약

현재 구현된 주요 화면:

- 로그인 화면
- 가족 생성 화면
- 부모 대시보드
- 부모 미션 관리 화면
- 부모 가족 관리 화면
- 새 미션 등록 화면
- 새 리워드 등록 화면
- 가족 멤버 추가 화면
- 아이 홈 화면
- 아이 상점 화면
- 아이 쿠폰함 화면
- 아이 프로필 화면
- 아이 포인트 수정 화면 UI 초안

현재 확인된 미완성 요소:

- 활동 통계 화면은 실제 데이터 기반 1차 구현 완료
- 부모 설정 화면은 placeholder 상태
- 부모 리워드 관리 화면은 목록/수정/삭제까지 연결 완료
- 아이별 포인트 표시 및 포인트 수정 API 연결 완료
- 미션 수정/삭제 기능 연결 완료
- 부모 대시보드의 "승인 내역 보기", "전체보기" 연결 완료
- 부모 대시보드의 "아이 모드 전환"은 아직 미구현
- 아이 프로필 화면은 실제 유저/가족/포인트/미션/쿠폰 데이터 기반으로 전환 완료
- 부모 설정 화면은 가족 정보/초대코드/계정 정보/로그아웃까지 연결 완료

## 2. 개발 원칙

- 한 번에 한 개의 사용자 흐름을 끝까지 완성한다.
- UI만 만들지 않고 API 연결과 상태 갱신까지 같이 마무리한다.
- 부모 앱과 아이 앱 모두 "사용자가 실제 눌렀을 때 동작하는 상태"를 기준으로 완료 처리한다.
- 새 기능 구현 후에는 관련 provider, repository, navigation까지 연결한다.

## 3. 우선순위별 개발 계획

### Phase 1. 포인트 기능 완성

상태: 완료

목표:

- 부모가 아이별 포인트를 조회하고 지급/차감할 수 있도록 완성
- 아이 화면에서 실제 포인트가 일관되게 표시되도록 정리

작업 항목:

1. 가족 관리 화면에서 아이별 현재 포인트를 실제 데이터로 표시
2. 아이 포인트 수정 화면에 `PointRepository.adjustPoints()` 연결
3. 포인트 수정 후 가족 관리 화면 / 아이 홈 / 아이 상점의 포인트 상태 갱신
4. 포인트 조정 사유 입력값을 API 요청에 포함
5. 에러/성공 메시지 UX 정리

완료 기준:

- 부모가 특정 아이의 포인트를 수정하면 즉시 화면에 반영된다.
- 아이 홈과 상점에서 동일한 포인트 잔액이 보인다.

### Phase 2. 활동 통계 화면 완성

상태: 완료

목표:

- 부모가 아이별 활동 현황을 확인할 수 있도록 통계 화면 구현

작업 항목:

1. `child_stats_screen.dart` 실제 화면 구현
2. 표시할 지표 정의
3. 백엔드에 필요한 통계 API가 없으면 우선 현재 데이터 기반 요약부터 구현
4. 미션 완료 수, 승인 대기 수, 누적 포인트 등 카드형 UI 구성
5. 가족 관리 화면에서 선택한 아이 정보를 통계 화면으로 전달

완료 기준:

- 가족 관리 화면에서 아이의 "활동 통계" 버튼을 누르면 실제 데이터가 보인다.

### Phase 3. 부모 리워드 관리 완성

상태: 완료

목표:

- 부모가 리워드를 생성하는 수준을 넘어서 조회/수정/삭제까지 할 수 있게 함

작업 항목:

1. `parent_store_screen.dart` 에 등록된 리워드 목록 표시
2. 리워드 수정 화면 또는 기존 생성 화면 재사용 방식 설계
3. 리워드 삭제 기능 추가
4. 활성/비활성 상태 관리 여부 결정
5. 리워드 구매 내역 부모 조회 필요 여부 검토

완료 기준:

- 부모가 리워드를 생성/조회/수정/삭제할 수 있다.

### Phase 4. 미션 관리 고도화

상태: 완료

목표:

- 미션 생성만 가능한 상태에서 실제 운영 가능한 관리 기능으로 확장

작업 항목:

1. 미션 수정 기능 구현
2. 미션 삭제 기능 구현
3. 부모 대시보드 "승인 내역 보기" 화면 추가
4. 부모 대시보드 "전체보기" 버튼 연결
5. 필요 시 미션 필터(아이별/상태별) 추가
6. 현재 `description` 에 문자열로 저장되는 수행대상/반복주기 구조 개선 검토

완료 기준:

- 부모가 생성한 미션을 다시 수정하거나 삭제할 수 있다.
- 승인 요청/승인 내역을 대시보드에서 탐색할 수 있다.

### Phase 5. 아이 프로필/설정 실데이터화

상태: 완료

목표:

- 아이 프로필 화면을 더미 데이터에서 실제 사용자 데이터 기반으로 전환

작업 항목:

1. 아이 이름/닉네임/아바타 실제 연동
2. 완료한 미션 수, 포인트 등 통계 실제 연동
3. 푸시 알림 설정 저장 방식 결정
4. "내 정보 수정" 화면 구현 여부 결정
5. "앱 설정" 화면 구현 여부 결정

완료 기준:

- 아이 프로필이 실제 계정/가족 데이터 기반으로 보인다.

### Phase 6. 부모 설정 화면 완성

상태: 완료

목표:

- 부모용 설정 화면을 운영 기능 중심으로 완성

작업 항목:

1. 현재 가족 정보 표시
2. 초대코드 확인/공유
3. 로그아웃
4. 계정 정보 확인
5. 앱 정보, 버전 정보, 고객문의 항목 필요 여부 검토

완료 기준:

- 설정 화면이 단순 placeholder 가 아니라 실제 관리 화면 역할을 한다.

## 4. 권장 실제 진행 순서

아래 순서대로 진행하는 것을 권장한다.

1. Phase 1: 포인트 기능 완성
2. Phase 2: 활동 통계 화면
3. Phase 3: 부모 리워드 관리
4. Phase 4: 미션 관리 고도화
5. Phase 5: 아이 프로필/설정 실데이터화
6. Phase 6: 부모 설정 화면 완성

이 순서를 추천하는 이유:

- 포인트는 부모/아이 양쪽 화면에 영향을 주는 핵심 데이터다.
- 활동 통계는 가족 관리 흐름과 자연스럽게 이어진다.
- 리워드와 미션은 현재 생성은 가능하므로 다음 단계로 확장하기 좋다.
- 프로필/설정은 핵심 운영 흐름보다 후순위다.

## 5. 다음 작업 시작점

현재 정의된 Phase 1 ~ Phase 6 구현은 모두 완료했다.

다음 작업은 신규 phase로 확장하면 된다.

권장 후속 작업:

1. 부모 대시보드의 "아이 모드 전환" 실제 계정/역할 전환 흐름 설계
2. 아이/부모 프로필 수정 기능 추가
3. 푸시 알림 설정 저장 API 또는 로컬 저장 구조 추가

## 6. 작업 방식

이 문서를 기준으로 앞으로는 다음 방식으로 진행한다.

- 한 phase 씩 구현
- phase 시작 전에 세부 작업 체크리스트 확정
- 구현 후 관련 화면 동작 여부 점검
- 완료 시 문서에 완료 상태 반영
